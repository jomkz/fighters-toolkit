# Terrain Map (.T2)

FA_2.LIB contains 16 `.T2` files — one per theater. Each stores the terrain grid data (height, color, surface type) for a campaign area. Referenced by `.MM` theater files via the `map` keyword (e.g. `map apa.T2`).

## Header Layout

```
Offset  Size  Description
------  ----  -----------
0x00     4    Magic: "BIT2" (ASCII, no null)
0x04    60    Theater name, null-padded ASCII (e.g. "Panama")
0x40    20    Unknown (observed all-zero)
0x54    12    PIC texture atlas reference, null-padded ASCII (e.g. "apa.PIC")
0x60     4    Unknown (observed 0)
0x64     4    u32 LE: map dimension (tile columns = tile rows; 25 or 32)
0x68     4    Unknown (observed 0)
0x6C     4    u32 LE: map dimension (duplicate of 0x64)
0x70     8    Unknown (observed all-zero)
0x78     4    u32 LE: always 2048
0x7C     4    u32 LE: stored_col_count × 256 (stored_col_count ≥ dim; may pad to even column)
0x80     ...  Terrain data payload (structure below)
```

## Map Sizes

`[0x7C]` encodes the **stored column count × 256** — which may exceed `dim` when the tile array is column-padded:

| Grid | Files | dim_x (`[0x64]`) | dim_y (byte `[0x7D]`) | tiles | File size |
|------|-------|-----------------|----------------------|-------|-----------|
| 32×32 | 11 | 32 | 32 | 1024 | 199,829 B |
| 25×26 | 4 (EGY/FRA/UKR/VLA) | 25 | 26 | 650 | 126,899 B |
| 25×25 | 1 (TVIET) | 25 | 25 | 625 | 122,024 B |

When addressing tile (col, row): `tile_index = row × dim_x + col`.

## Data Payload

Payload at offset `0x80` = **21-byte sub-header** + **N_tiles × 195 bytes** of tile data.

```
Offset (from 0x80)  Size  Description
------------------  ----  -----------
0                   21    Sub-header (partially decoded; see below)
21                  195   Tile 0
216                 195   Tile 1
...                       (tiles stored row-major; tile_index = row × stored_cols + col)
```

### Sub-Header (21 bytes)

| Byte(s) | Observed values | Notes |
|---------|-----------------|-------|
| 0 | `0x00` | Always 0 |
| 1 | `0x20` (32×32), `0x19` (25×N) | Map x-dimension (= `dim` at header `0x64`) |
| 2–3 | `0x00 0x00` | Always zero |
| 4–16 | *class constant — see below* | Not per-theater; one fixed value per grid-size class |
| 17 | `0x95` | Always 0x95 = 149 = absolute file offset of first tile (0x80 + 21) |
| 18–20 | `0x00 0x00 0x00` | Always zero |

Bytes 4–16 are **class constants** — all theaters of the same grid-size class share identical values. Full survey of all 16 theaters confirmed three distinct values:

| Grid class | Files | Bytes 4–16 (hex) |
|------------|-------|-------------------|
| 32×32 | APA BAL CUB GRE IRA KURILE LFA NSK PGU SPA WTA (11 files) | `00 95 00 03 00 00 01 00 00 00 01 00 00` |
| 25×26 | EGY FRA UKR VLA (4 files) | `00 15 E8 01 00 D0 00 00 00 C8 00 00 00` |
| 25×25 | TVIET (1 file) | `00 55 D5 01 00 C8 00 00 00 C8 00 00 00` |

These values do not map cleanly to tile counts, file sizes, or simple dim multiples from binary analysis alone. World-space scale or coordinate bounds likely — requires Ghidra trace to decode.

### Tile Layout (195 bytes = 65 × 3-byte records)

Each tile's 195 bytes divide into **65 consecutive 3-byte records**:

| Record index | Role |
|--------------|------|
| 0 | Tile summary — representative sample for LOD/overview rendering (same structure as sub-records) |
| 1–64 | 8×8 sub-tile grid, row-major (rec 1 = sub-row 0 col 0; rec 8 = sub-row 0 col 7; rec 9 = sub-row 1 col 0; …) |

The summary record is **not** the NW corner (sub-record 1) and is **not** the dominant surface class — survey of mixed-terrain tiles in UKR.T2 shows disagreement in both cases. Exact selection algorithm (LOD sampling, weighted average, or engine-computed) requires Ghidra trace.

### 3-Byte Sub-Tile Record

```
Byte  Field              Values / Notes
----  -----              -----
0     Surface class      0xFF = water/ocean (all theaters)
                         0xD2 = land (APA, IRA, GRE, NSK, LFA — single land type)
                         0xD0–0xDA = multiple land classes (EGY, UKR: sand, rocky, dunes, etc.)
                         0xC2, 0xC4–0xC7 = land classes (TVIET: jungle, paddy, savanna, etc.)
1     Elevation band     0 = sea level / coastal; higher = altitude above sea
                         Max observed per theater: APA=12, GRE=4, IRA=6, EGY=6,
                         NSK=12, LFA=4, UKR=6, TVIET=16
2     Texture variant    0–31; selects a sub-texture within the PIC atlas for this surface class
```

**Relation to `.MM` tmap coordinates:**

`MISSIONTextProc` is the MM file text parser (keyword dispatch loop). The `tmap` keyword handler (confirmed from decompile) reads **4 × s16** values per entry (format `tmap %d %d %d %d`), bounded to 3500 entries (counter `tlistSize`), stored into arrays at `tlist`/`DAT_005733e4`. A second form `tmap_named` reads a name string + 2 × s16, stored as a 53-byte struct at `tdic + i × 0x35`. The `tdic` keyword reads 1 × u32 + 32 × u8 into the same struct at offset `+0x0D` (max 300 entries, counter `tdicSize`).

Approximate tile-index formula (based on observed `tmap` values in MM files):

```
T2_tile_col = tmap_col / 8
T2_tile_row = tmap_row / 8
tile_index  = T2_tile_row × dim_x + T2_tile_col
```
