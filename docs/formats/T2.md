# Terrain Map (.T2)

FA_2.LIB contains 16 `.T2` files — one per theater. Each stores the terrain grid data (height, color, surface type) for a campaign area. Referenced by `.MM` theater files via the `map` keyword (e.g. `map apa.T2`).

## Header Layout

```
Offset  Size  Description
------  ----  -----------
0x00     4    Magic: "BIT2" (ASCII, no null)
0x04    60    Theater name, null-padded ASCII (e.g. "Panama")
0x40    20    Unknown (observed all-zero)
0x54    12    PIC texture atlas reference, null-padded ASCII (e.g. "apa.PIC")
0x60     4    Unknown (observed 0)
0x64     4    u32 LE: map dimension (tile columns = tile rows; 25 or 32)
0x68     4    Unknown (observed 0)
0x6C     4    u32 LE: map dimension (duplicate of 0x64)
0x70     8    Unknown (observed all-zero)
0x78     4    u32 LE: always 2048
0x7C     4    u32 LE: derived from dimension (see table)
0x80     ...  Terrain data payload (structure below)
```

## Map Sizes

Two grid sizes are in use across all 16 theaters:

| Grid | Files | [0x64]/[0x6C] | [0x78] | [0x7C] | Total size |
|------|-------|---------------|--------|--------|------------|
| 32×32 | 11 | 32 | 2048 | 8192 | 199,829 bytes |
| 25×25 | 4 | 25 | 2048 | 6656 | 126,899 bytes |
| 25×25 (variant) | 1 (Vietnam) | 25 | 2048 | 6400 | 122,024 bytes |

The PIC reference names the texture atlas used for terrain tile rendering. Each theater has a matching `.PIC` file with the same base name (e.g. `APA.T2` → `apa.PIC`).

## Data Payload

Structure at offset 0x80 is not yet decoded. The terrain data encodes height, color index, and surface type per tile. Given the payload sizes (~199 KB for 32×32, ~127 KB for 25×25), per-tile data is approximately **190 bytes** per tile for 32×32 (1024 tiles) — suggesting the payload includes not just tile indices but sub-tile data or a heightmap at higher resolution than the tile grid.
